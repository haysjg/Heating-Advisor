Règles de recommandation

Tempo ROUGE  Toujours le poêle
En jour Tempo Rouge, le kWh en Heures Pleines coûte {{ "%.4f"|format(config.TEMPO_PRICES.RED.HP) }} €. La clim n'est jamais recommandée, même si son COP est favorable.
Exemple : Tempo Rouge, 3°C dehors, COP clim estimé à 2,0 → coût clim ≈ 0,38 €/kWh thermique vs poêle ≈ 0,047 €/kWh thermique. → Poêle recommandé.
Seuil de confort clim  Poêle si T° ext < {{ config.CLIM.comfort_min_temp }}°C
En dessous de {{ config.CLIM.comfort_min_temp }}°C extérieurs, la clim ({{ config.CLIM.model }}) chauffe insuffisamment : le flux d'air chaud est trop faible pour maintenir le confort. Le poêle est recommandé à sa place, même si le coût électrique serait théoriquement inférieur.
Exemple : 5°C dehors, Tempo Blanc → coût clim ≈ 0,086 €/kWh thermique vs poêle ≈ 0,047 €/kWh thermique. Mais clim insuffisante sous {{ config.CLIM.comfort_min_temp }}°C → Poêle recommandé.
Limite technique clim  Clim arrêtée si T° ext < {{ config.CLIM.min_outdoor_temp }}°C
En dessous de {{ config.CLIM.min_outdoor_temp }}°C extérieurs, la clim ne fonctionne plus. Seul le poêle est proposé.
Comparaison économique  Clim si coût kWh < coût granulés
Hors cas particuliers ci-dessus, l'app compare le coût réel par kWh thermique des deux systèmes en tenant compte du COP de la clim (variable selon T° ext) et du rendement du poêle.
Exemple : 10°C dehors, Tempo Blanc HP ({{ "%.4f"|format(config.TEMPO_PRICES.WHITE.HP) }} €/kWh), COP clim estimé à {{ config.CLIM.nominal_cop }} → coût clim ≈ {{ "%.3f"|format(config.TEMPO_PRICES.WHITE.HP / config.CLIM.nominal_cop) }} €/kWh thermique vs poêle ≈ {{ "%.3f"|format(config.POELE.pellet_price_per_kg / config.POELE.consumption_kg_per_hour / config.POELE.thermal_output_kw) }} €/kWh thermique. → Clim recommandée.
Nuit  Pas de chauffage en Heures Creuses (22h – 6h)
Entre {{ config.HP_END }}h et {{ config.HP_START }}h (Heures Creuses), aucun chauffage n'est recommandé, quelle que soit la température.

Thermostat automatique

Seuils d'allumage / extinction
Le thermostat allume le chauffage quand la température intérieure passe sous {{ config.THERMOSTAT.temp_on }}°C (ressenti) et l'éteint quand elle atteint {{ config.THERMOSTAT.temp_off }}°C. Le ressenti tient compte de l'humidité (correction de ±{{ config.THERMOSTAT.humidity_correction_factor }}°C par % d'écart à {{ config.THERMOSTAT.humidity_reference|int }}% HR).
Exemple : 19,8°C réel, 62% HR → ressenti ≈ 19,2°C < {{ config.THERMOSTAT.temp_on }}°C → allumage.
Durée minimale avant extinction sur température
Pour éviter les cycles courts, le chauffage ne peut pas s'éteindre avant {{ config.THERMOSTAT.min_on_minutes }} min (poêle) ou {{ config.THERMOSTAT.min_on_minutes_clim }} min (clim) après son allumage, même si la température cible est déjà atteinte.
Plages horaires
Le thermostat ne fonctionne qu'à l'intérieur des plages horaires configurées dans Config. Hors plage, le système s'éteint après une grâce de {{ config.THERMOSTAT.end_of_schedule_grace_minutes }} min (pour absorber les démarrages tardifs en fin de plage).
Suspension après extinction manuelle
Si le poêle ou la clim est éteint manuellement, le thermostat se suspend pendant {{ config.THERMOSTAT.manual_off_suspend_hours }}h pour ne pas le rallumer immédiatement. Il reprend ensuite normalement. Rallumer manuellement annule la suspension.
Exemple : tu éteins le poêle à 21h → thermostat suspendu jusqu'à 1h, même si la température redescend sous {{ config.THERMOSTAT.temp_on }}°C.
Transition clim → poêle : chevauchement {{ config.THERMOSTAT.clim_to_poele_overlap_minutes }} min
Quand la recommandation passe de la clim au poêle, le poêle est allumé immédiatement mais la clim reste active pendant {{ config.THERMOSTAT.clim_to_poele_overlap_minutes }} min le temps que le poêle monte en température.
Exemple : recommandation passe à poêle à 8h00 → poêle allumé à 8h00, clim éteinte à 8h20.

Mode présence (zones GPS)

Trois zones
home Dans la maison (rayon ~350 m) — thermostat actif normalement.

nearby En zone de proximité (~3500 m) — allumage bloqué après {{ config.THERMOSTAT.nearby_no_ignition_after }}h, extinction après {{ config.THERMOSTAT.nearby_grace_minutes }} min de grâce.

away Absent — extinction après {{ config.THERMOSTAT.away_grace_minutes }} min de grâce.
Exemple : tu pars faire une course à 19h30, GPS en zone away. Grâce de {{ config.THERMOSTAT.away_grace_minutes }} min → extinction à 19h45 si tu n'es pas rentré. Si tu rentres à 19h40, la grâce se réinitialise et le chauffage continue.
Grâce GPS
Les {{ config.THERMOSTAT.away_grace_minutes }} min de grâce avant extinction absorbent les fluctuations GPS (passage temporaire hors zone sans déplacement réel). La grâce repart à zéro dès que la zone redevient nearby ou home.
Exemple : GPS fluctue brièvement hors zone à 14h02, rentre dans nearby à 14h05 → grâce de {{ config.THERMOSTAT.away_grace_minutes }} min jamais écoulée → aucune extinction.

Commandes utiles (NAS Synology)

Accéder aux logs (SSH sur le NAS)
# Logs en direct
cd /volume1/docker/heating-advisor
docker-compose logs -f

# Logs filtrés sur le thermostat
docker-compose logs -f | grep -i thermostat

# Logs des dernières 2 heures
docker-compose logs --since 2h

# Logs depuis une date / heure précise
docker-compose logs --since "2026-04-13T06:00:00"
Les logs de niveau DEBUG (états HA bruts) ne s'affichent pas par défaut. Les décisions importantes (allumage, extinction, présence, zones) sont en niveau INFO.
Mettre à jour et rebuilder le container
# Depuis le NAS, en SSH
cd /volume1/docker/heating-advisor
wget https://github.com/haysjg/Heating-Advisor/archive/refs/heads/main.tar.gz -O ha.tar.gz
tar xzf ha.tar.gz --strip-components=1 && rm ha.tar.gz
docker-compose down
docker-compose build --no-cache
docker-compose up -d
Le --no-cache est indispensable : les templates HTML sont copiés dans l'image au moment du build, pas montés en volume.
Vérifier l'état du container
docker-compose ps
docker-compose logs --tail=20
